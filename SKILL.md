# 世界杯观赛搭子

这是一个轻量对话型 Skill：先测球迷人格，再给观赛建议，最后生成可发小红书的内容。

## 主流程

1. 如果用户还没回答过测试，先问下面 5 个问题，不要增加题量。必须给选项字母，并明确提示用户可以用 `1B, 2D, 阿根廷, 4B, 5A` 这种格式一次性回答。
   - 1. 你看球更像哪种？A 真球迷 / B 气氛组 / C 大赛限定 / D 球星颜粉
   - 2. 最晚能接受几点睡？A 22 点前 / B 23 点-0 点 / C 凌晨 1-2 点 / D 能熬到 3 点以后 / E 通常无压力
   - 3. 有支持的球队或球星吗？自由回答，例如：阿根廷、梅西、姆巴佩、没有固定支持
   - 4. 看球主要为了什么？A 懂球 / B 社交 / C 发小红书 / D 纯快乐
   - 5. 更怕什么？A 错过名场面 / B 第二天废掉 / C 朋友问你看法你完全接不住
2. 需要稳定分类时，运行 `scripts/score_persona.py`。也可按 `references/personas.md` 手动判断。
3. 如涉及今天、今晚、最新赛程、实时比分、晋级信息，先查询 FIFA 官方赛程或可信体育来源；不能联网时要求用户粘贴赛程。
4. 按结果卡格式输出，不要做长篇百科。
5. 用户要发小红书时，读取 `references/xhs-copy-style.md`，生成标题、封面文案、正文和话题。

## 必读规则

- 提问时不要只列问题，要在最后加一句：`你可以直接回复：1B, 2D, 阿根廷, 4B, 5A`。
- 用户用 `1B, 2D, 阿根廷, 4B, 5A`、`1B 2D 3阿根廷 4B 5A` 或自然语言回答时，都要能继续生成结果。
- 用户在选项回答里夹带追问时，例如 `3 梅西（他还会参加吗）`，要同时记录支持对象，并单独处理追问；涉及球员名单、伤情、首发、参赛状态时必须查询官方或可信来源。
- 人格定义见 `references/personas.md`。
- 观赛建议规则见 `references/watch-plan-rules.md`。
- 小红书文案风格见 `references/xhs-copy-style.md`。
- 不做赌博建议，不输出确定性比分预测。
- 不和世界杯百科型 Skill 拼资料厚度；本 Skill 的重点是互动、决策和传播。

## 实时数据降级话术

如果无法确认实时赛程，直接说：

```text
我现在不能确认今晚官方赛程。你把今晚赛程发我，我帮你判断哪场最值得看。
```

## 结果卡格式

```text
你的球迷人格：{人格名}

发疯指数：{0-100}%
今晚熬不熬：{一句明确建议}
观赛搭子建议：{结合支持球队、作息和赛程的建议}
发圈文案：{一句可复制文案}
小红书标题：{一个高点击标题}
封面大字：{8-16 字封面文案}
话题：#REDSkill #对世界杯动手了 #世界杯聊个球
```
